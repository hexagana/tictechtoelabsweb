Build & Customise a Department SharePoint Site

You'll create a team site for your department, then add the four building blocks that turn a blank page into a working hub: a list, a task board, quick-link buttons, and a document library.

🎯 What you'll build today

  • Create a department team site and understand how a SharePoint page is edited.
  • Add a SharePoint list to track structured information (e.g. a request or asset register).
  • Add a Planner / To-Do task board so the team can see who's doing what.
  • Add a section of quick-link buttons to your most-used tools and pages.
  • Integrate a document library right onto the home page — connected to the folder system from Topic 1.

🧱 One skill unlocks all four: adding a web part

Everything below is added the same way. On your site's home page you click Edit (top-right), hover where you want the content, click the circled +, and pick a web part. Learn this once and you can add anything.

0

Create the department site

Your team's home base in SharePoint
~7 min
Objective: Spin up a Team site for your department that everyone can access, with its own document library ready to use.

Open SharePoint

Go to office.com (or the Microsoft 365 app launcher — the ▦ waffle, top-left) and click SharePoint.

Click "＋ Create site"

You'll see it near the top of the SharePoint start page. Choose Team site — this gives your department shared files, a calendar, and a document library out of the box.

Name it after the department

e.g. Afton – Operations. SharePoint auto-creates the address and a matching Microsoft 365 group.

Add members

Add your team as Members (can edit) and keep yourself as Owner (full control). Click Finish.

1

Add a SharePoint list

Track structured information — like a smart, shared spreadsheet
~9 min
Objective: Create a list to track a repeating process for the department — for example an Equipment / Asset Register or a Request Tracker — with columns, statuses, and views everyone shares.

On the home page, click Edit → ＋ → List

Or from the command bar choose ＋ New → List. You can start Blank, from an Excel file, or from a ready-made template.

Name the list

e.g. Equipment Register. Give it a clear description so the team knows what it's for.

Add your columns

Click ＋ Add column and choose a data type per column — Text, Choice (for status), Date, Person, Number. Data types keep entries clean and filterable.

Enter a few rows, then save a view

Add items with ＋ New. Filter or group by Status, then Save view as "Open items" so the team lands on the right filter.

Republish the page

Back on the home page, the list web part now shows live data. Click Republish to make it visible to everyone.

Tip: A list beats a spreadsheet on a shared drive because two people can edit at once, every change is tracked, and you can filter, group, and even trigger reminders — no version conflicts.
2

Add a Planner / To-Do task board

See who's doing what, at a glance
~9 min
Objective: Embed a Planner board on the site so the department can assign tasks, set due dates, and track progress across simple columns (buckets).

Edit the page → ＋ → search "Planner"

In the web part picker, type Planner and select the Tasks by Planner and To Do web part.

Create a new plan (or link an existing one)

Choose Create a new plan and name it — e.g. Operations – Weekly Tasks. It's automatically tied to your team.

Set up buckets

Buckets are your columns. Create To do, In progress, and Done — or use stages that fit your workflow.

Add tasks and assign people

Click ＋ Add task, give it a title, a due date, and assign a team member. Drag tasks between buckets as work progresses.

Republish

The board is now live on the home page and stays in sync with Planner and Microsoft To Do.

Tip: Tasks assigned in Planner also appear in each person's Microsoft To Do and Teams — so nobody has to remember to check the site.
3

Add a section of quick-link buttons

One-click access to the tools your team uses daily
~7 min
Objective: Add a Quick links web part — a tidy row of clickable buttons pointing to your most-used pages, tools, and documents.

Edit the page → ＋ → Quick links

In the picker, choose Quick links. It drops in a labelled section you can fill with buttons.

Click ＋ Add and paste a link

Paste a URL (an internal SharePoint page, a document, or an external site like your ERP), give it a friendly title, and pick an icon.

Choose a layout

In the web part settings (pencil ✎), pick a layout — Button, Compact, or Filmstrip. Button and Compact look cleanest for a link bar.

Reorder & republish

Drag buttons into priority order (most-used first) and click Republish.

4

Integrate a document library

Surface your files right on the home page
~9 min
Objective: Show a document library directly on the department home page — organised with the date-first folder convention from Topic 1 — so files are visible without hunting through menus.

Set up the library first

Open Documents and create your top-level folders using PARA + the date-first convention: Projects, Areas, Resources, Archives, with dated folders inside (e.g. 2026-07-16 Afton Onboarding).

Edit the home page → ＋ → Document library

In the picker, choose the Document library web part.

Pick which library and view to show

Select your Documents library, then choose a view (e.g. "All Documents" or a custom filtered view). You can show a specific folder as the starting point.

Set the size & command bar

In settings (✎), choose how many items to show and whether people can upload directly from the home page. Turn on the command bar so they can add files in place.

Republish

Files now live on the home page. Because folders are date-first, they stay in order automatically — the payoff from Topic 1.

Tip: Click Sync (or "Add shortcut to OneDrive") and the library appears in File Explorer — so the team can work with SharePoint files as if they were local, while everything stays backed up and shared.

✅ You've built a working department hub

List

Structured, filterable tracking that beats a shared spreadsheet.

Planner

A shared task board synced to To Do and Teams.

Quick links

One-click access to the team's daily tools.

Library

Files on the home page, self-sorting by date.
